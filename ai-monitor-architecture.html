PID 协调层

所有组件通过进程 ID 建立协作关系

Agent ⇄ 监督脚本 ⇄ 启动脚本 均以 PID 为纽带；提示词发送、结果返回均通过 PID 关联

Agent PID 接收监督脚本轮询与继续工作提示词；持有目标脚本 PID 进行持续监控。
启动脚本 PID 启动命令生成（如 python app.py —pid=xxx），是 Agent 监控的核心目标进程。
监督脚本 PID 持有自身 PID；通过 Agent PID 发送轮询 + 提示词；接收结果回传。
并发任务 PID 多线程 / 子进程各自注册 PID，Agent 可逐一追踪与控制。

启动顺序

用户选择工作区与 Agent → 设置或生成 PID → 执行启动命令（如 python app.py —pid=xxx）→ Agent 启动并接收提示词 → 监督脚本按配置间隔开始轮询。

监督逻辑（轮询制）

监督脚本每 N 秒（用户可配）检查 Agent 进程是否处于工作状态。若 Agent 空闲或停止，则通过 Agent PID 发送"继续监控目标脚本 PID"的提示词，确保 Agent 持续工作。

提示词边界 — 非代码连接

Agent 与启动脚本之间不是代码接口调用。Agent 通过提示词获知目标脚本的 PID、预期行为与监控要求。所有交互（定位、监控、修改、结果回传）均走提示词通道。
